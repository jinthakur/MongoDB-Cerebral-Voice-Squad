8. Problem Statement & Example Projects
All projects must be built in one of our three problem statements outlined below!
Statement One: Codebase Context Optimization
Todayâ€™s AI coding agents struggle to understand large, evolving codebases. How can we build tools that give models persistent, semantic awareness of an entire codebase? How can we allow them to reason across files, adapt to organizational conventions, and generate contextually accurate, maintainable code at scale?
Examples:
Advanced RAG systems that can traverse static and dynamic codebase analyzer outputs, as well as structures like ASTs (Abstract Syntax Trees).
Dynamic context management in large codebases to manage long-running tasks in large open-source projects such as Apache Kafka, Kubernetes, etc.
Statement Two: Reinforcement Learning & Post Training
Current improvements often rely on in-context learning. How can we build comprehensive toolchains that automate the RL lifecycle: from creating coding-specific environments and automated reward signals to optimizing hyperparameters? How do we use these tools to execute advanced post-training strategies, permanently refining model policies and weights to solve complex, multi-step engineering problems?
Examples:
RL Environment Generators that can create gym-like sandboxes and harnesses
Automated Reward Modeling tools that use compiler feedback, runtime execution success, and static analysis to create scalar reward signals for RLHF (Reinforcement Learning from Human Feedback) or RLAIF (AI Feedback).
Hyperparameter Optimization Suites: Specialized tooling for tuning the learning rates and policy update constraints specifically for code-generation models to prevent "forgetting" basic syntax while learning new logic.
Statement Three: Collaborative Code Generation Tooling
Coding with AI shouldnâ€™t be a solo activity. How can we build collaborative environments where humans and AI co-develop, review, and deploy software together in real time, supporting both human-to-human and human-to-AI interactions? How can AI tools integrate seamlessly into existing team workflows to optimize live collaboration and collective problem-solving?
Examples:
Automatic verification and testing of codebase changes to iterate to a result such as visual verification of website changes.
Multi-agent collaboration and human-agent collaboration at the same time, a team of agents and humans interacting with each other to make code changes and solve PRs.
Automatic debugging tools that can run step-wise through projects, add breakpoints, and solve issues.
